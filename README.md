  - 취미: AI 개발자

## 소개
- Backend Developer with Frontend..?
  - 담당 중인 시스템의 프론트와 백 영역이 제대로 분리되어 있지 않아 영역과 상관없이 개발
- 급여 아웃소싱을 진행하는 회사에서 급여 담당자와 고객사에서 발생하는 문의 & 오류를 처리함
- 시스템 개선을 위한 요구사항이 발생하면 개발
  - 요구사항 분석, 설계서 작성, 개발, 매뉴얼 작성, 프로젝트 발표 순 진행
- 회사 내 신규 프로젝트를 타부서와 협업을 통해 진행
- 소득정산 모듈의 개선을 위해 TFT에 참석해 개선 요구사항 개발

### 가치관
- 코드 소스와 다양한 IT지식을 모두와 자유롭게 공유하는 것, 지식의 습득 자유를 실천하고자 함
  - 장애 발생마다 영어로 검색해야 해결이 쉽다는 사실이 안타까움
- 다양한 경험을 하는 것을 즐기며 오래된 기술도 필요에 따라 사용해야 함
  - 모든 기술은 상황에 맞게 사용해야 한다고 생각함
- 시스템에서 오류가 발견되지 않았더라도 멀쩡한 것은 아니다
  - 오래된 프로그램이 안정적인 것처럼 보여도 완전/안전하다고 판단하면 안됨
- 잘못된 부분을 선제적으로 파악하여 조치하는 문화를 지향함
